# Практическая работа: подсчёт автомобилей и оценка скорости по видео

## Цель

Разработать программу, которая обнаруживает и сопровождает автомобили на видео, считает уникальные машины, пересёкшие контрольную линию, и приближённо оценивает их скорость.

Исходные данные:

- видео `ScreenRecorderProject49.mp4`;
- модель `yolo26s.pt`;
- размер входа модели `imgsz=1280`;
- анализируемый класс COCO: `car` (ID 2).

## Шаг 1. Подготовить окружение

1. Установите Python, `ultralytics` и `opencv-python`.
2. Проверьте, что видео открывается через `cv2.VideoCapture`.
3. Получите FPS, число кадров, ширину и высоту видео.
4. Загрузите модель командой `YOLO("yolo26s.pt")`.

## Шаг 2. Выполнить детекцию автомобилей

1. Передавайте кадры модели с параметрами `imgsz=1280`, `classes=[2]` и, например, `conf=0.25`.
2. На каждом кадре получите координаты рамок автомобилей.
3. В качестве точки положения автомобиля используйте нижний центр рамки: `((x1+x2)/2, y2)`. Для дорожной сцены эта точка обычно ближе к месту контакта машины с дорогой, чем геометрический центр рамки.
4. Нарисуйте рамки и confidence на кадре.

## Шаг 3. Добавить трекинг

Простое суммирование рамок по кадрам неверно: одна машина будет посчитана много раз. Используйте трекер ByteTrack или BoT-SORT:

```python
result = model.track(
    frame,
    persist=True,
    tracker="bytetrack.yaml",
    imgsz=1280,
    classes=[2],
    conf=0.25,
    verbose=False,
)[0]
```

Для каждой рамки сохраните `track_id`, текущую точку автомобиля и несколько предыдущих точек его траектории.

## Шаг 4. Задать линию подсчёта

1. Выберите на дороге контрольную линию, поперечную направлению движения.
2. Задайте её двумя точками `P1(x1, y1)` и `P2(x2, y2)`.
3. Для каждой траектории определяйте, по какую сторону линии находилась точка автомобиля на прошлом и текущем кадре.
4. Смена стороны означает пересечение линии. Дополнительно проверьте, что перемещение происходит в нужном направлении.

Знак стороны точки `P(x, y)` относительно направленной линии можно вычислить так:

```python
side = (p2_x - p1_x) * (y - p1_y) - (p2_y - p1_y) * (x - p1_x)
crossed = previous_side * current_side < 0
```

## Шаг 5. Подготовить измерение скорости

По одной линии скорость определить нельзя. Нужны две линии `A` и `B`, между которыми заранее известно реальное расстояние `S` в метрах.

1. Разместите линии поперёк одной полосы движения.
2. Измерьте реальное расстояние между ними по карте или по объекту известного размера (марка машины и её длина).
3. Для каждого `track_id` запишите время пересечения первой линии `t_A` и второй линии `t_B`.
4. Время кадра вычисляйте как `frame_index / FPS`.
5. Рассчитайте скорость:

```text
delta_t = t_B - t_A
v_m_s = S / delta_t
v_km_h = 3.6 * v_m_s
```

Из-за перспективы нельзя переводить пиксели в метры одним постоянным коэффициентом для всего кадра. Более точный вариант дополнительной части — построить гомографию плоскости дороги минимум по четырём опорным точкам.

## Пример расчёта

Пусть расстояние между линиями равно `S = 12 м`. Машина с `track_id=17` пересекла первую линию в `t_A = 10,4 с`, вторую — в `t_B = 11,3 с`.

```text
delta_t = 11,3 - 10,4 = 0,9 с
v = 12 / 0,9 = 13,33 м/с
v = 13,33 * 3,6 ≈ 48 км/ч
```

В отчёте эту величину следует назвать приблизительной и указать источник погрешности: неточность расстояния, положение линий, перспектива, пропуски детектора и смена `track_id`.

## Шаг 6. Сформировать итоговое видео

На каждом кадре отобразите:

1. рамку, класс, confidence и `track_id` автомобиля;
2. траекторию автомобиля;
3. линию подсчёта и две линии измерения скорости;
4. общее число пересёкших линию машин;
5. направление движения;
6. последнюю или текущую оценку скорости рядом с машиной.

Сохраните видео с исходным FPS и разрешением. Запишите отдельный CSV со столбцами:

```text
track_id,direction,crossing_time,t_A,t_B,delta_t,distance_m,speed_km_h
```

## Шаг 7. Проверить качество

1. Вручную посчитайте машины на небольшом фрагменте видео.
2. Сравните ручной и автоматический результаты.
3. Для 3–5 автомобилей вручную отметьте моменты пересечения линий и пересчитайте скорость.
5. Объясните минимум три причины ошибок и предложите способы уменьшения каждой ошибки.

4. Краткий отчёт: координаты линий, реальное расстояние, параметры модели и трекера, итоговое число машин, средняя скорость, ручная проверка и анализ ошибок.
